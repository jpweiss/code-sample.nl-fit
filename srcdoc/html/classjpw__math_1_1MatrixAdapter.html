Detailed Description

template<class S, template< class > class STOR_POL = policy::SequenceStorage>
class jpw_math::MatrixAdapter< S, STOR_POL >

Class MatrixAdapter.

A Adapter class for accessing a 1-D sequence as if it were a 2-D matrix.

It is designed for efficient access, not super-flexibility. The indices are 0-based, and cannot be altered. The leading index selects the row, i.e. indexing is C-style. You cannot change it to FORTRAN style. If you need features like that, see the boost library for more powerful classes. (Those classes are, however, also more slow.)

To use this class, you must also #include "details/MatrixAdapter.tcc" in the translation unit instantiating the MatrixAdapter<...> template.

The template argument, S, can be any form of sequence class or pointer to POD type. S must, at minimum, define an operator[].

To minimize assumptions on S, there are other template arguments, for specifying policy classes.

The remaining template arguments are all policy classes.

STOR_POL describes what kind of behaviors the storage type, S, guarantees. Usually, you'll use the default, policy::SequenceStorage. Specialize this class or use it as a model if you need to "roll your own."

This class internally uses the traits::ContainerType class template to determine the data type stored in S. If you have custom requirements, specialize traits::ContainerType for your needs.

Constructor & Destructor Documentation

MatrixAdapter ( )

Default Constructor.

Creates a 0x0 matrix.

Clearly such an object is useless. Don't even think about trying to use operator[] or at() with it.

All other constructors are guaranteed to create matrix objects with nonzero dimensions. Thus, you can uniquely identify any default-constructed instances of this class.

To "insert" data into a default-constructed instance of a MatrixAdapter, you can:

  1. Use operator=() to copy an existing MatrixAdapter into it. You don't want to do this, however: 
        typedef MatrixAdapter<somesequence_t> matrix_t;
    matrix_t mt;
    :
    :
    mt = matrix_t(42, 24);
    It won't perform as well as this:
  2. Use swap() with a temporary MatrixAdapter instance. This is most useful for "copy-constructing into" the empty matrix, like so: 
        typedef MatrixAdapter<somesequence_t> matrix_t;
    matrix_t mt;
    :
    :
    matrix_t nuDim_tmp(42, 24);
    mt.swap(nuDim_tmp);          // <-- Fast!

INPUT_ITERATOR  end,
size_t  n_rows,

)

Conversion/Data-Loading constructor.

Creates a new matrix using the data in the sequence defined by begin and end by "reshaping" it to the specified dimensions.

The new reshaped matrix satisfies 'm[i][j] = v_m[i*m.nColumns()+j]', where v_m is the sequence bound by begin and end.

Note that this c'tor should not be used when S is a POD-pointer array type. It will fail to compile if you try to call it.

const_vector_type_reference as_1D ( ) const [inline]

Returns the underlying 1-D data.

This MatrixAdapter class organizes its elements in "typical C fashion". That is, the indices are 0-based, and the last index (the columns) vary the fastest. Element access of the form m[i][j] is, therefore, equivalent to indexing v_m[i*m.nColumns()+j] on the underlying 1-D container, v_m .

There are several instances where you will wish to or even need to access the underlying 1-D container object. You can do so here ... but only for non-modifying operations.

RowProxy operator[] ( size_t  row_idx) [inline]

Indexes a row of the MatrixAdapter.

This operator allows you to write expressions like: 

 MatrixAdapter<vector<double> > m = some_MatrixAdapter_operation();
 m[2][7] = 3.141;

Note that the indices are all 0-based.

It returns an intermediate type that is not meant to be used directly. In fact, it's not meant to be seen at all, but (hopefully) will be optimized away by the compiler.
