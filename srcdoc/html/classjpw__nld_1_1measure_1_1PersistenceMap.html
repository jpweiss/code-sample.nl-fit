Detailed Description

A container class for a persistence map and its related data.

The main reason that it's a class and not a struct, with all member fields public, is to keep the m__phases and m__lags matrices properly-dimensioned relative to the m__map matrix. Furthermore, any time the m__map member is modified, m__phases and m__lags are updated accordingly.

Uses the compiler-generated copy-c'tor and assignment operator.

Inherits measure-related constants from BarrierMeasure

Note: At present, the internal algorithms for computing the persistence assume that you are generating a square map. If you call the c'tor, wipe() or clear() functions with a number of columns different from the number of rows, you won't be able to call computeCSStdDev() or computePersistence(). Ditto if you pass a non-square matrix to swap_map().

void clear ( size_t  n_rows,
size_t  n_columns = 0 
) [inline]

Calls Matrix::clear on the internal Matrix objects.

Unlike wipe(), this fn. "clears" the two axes Matrix objects by invoking fillAxes().

Unlike Matrix::clear, the dimension argument n_rows is required. The argument n_columns is optional; if omitted or 0, it will be set to n_rows.

After calling this function, both emptyCSStdDev() and empty() will return false. You'll need to rerun computePersistence() [or computeCSStdDev()].

Unless you are resizing the PersistenceMap object, this is not the function you are looking for. Use wipe() instead.

bool empty ( ) const [inline]

Check if the internal Matrix objects are "empty.".

The internal Matrix objects are never truly empty, as they have predetermined dimensions and, therefore, preallocated elements.

Instead, this function determines whether or not computeCSStdDev() and computePersistence() have ever been called.

When this function returns true, the functions as_1D(), data(), csAverage(), and csStdDev() will return meaninless data.
