Detailed Description

template<class FIT_FN, class DATA_T, unsigned POPULATION_SIZE = 40, unsigned N_GENERATIONS = 100, unsigned N_MUTATE = static_cast<unsigned>(POPULATION_SIZE*0.3), unsigned N_ELITE = 3, unsigned WIDE_XOVER_WEIGHT_X_1000 = 1490>
class jpw_nld::optimize::FitGA< FIT_FN, DATA_T, POPULATION_SIZE, N_GENERATIONS, N_MUTATE, N_ELITE, WIDE_XOVER_WEIGHT_X_1000 >

Optimize a function via Genetic Algorithms.

The template parameter FIT_FN must provide the following public member constants and functions:

  • static const index_t N_PARAMETERS
    The number of tunable parameters in the fit function. This is the vector whose value you're trying to find by minimization.
  • double chiSquared(const DATA_T& data, const dvector_t& params)
    Compute and return the error between the data and your function, evaluated at the parameters params.
  • static void randomParams(dvector_t& params)
    Fill the dvector_t with "constrained" random values, based on the model. The "constrained" random values should be uniformly distributed over a range sensible for each parameter. (E.g. if parameter #2 can't be smaller than -3, don't use an aribtrary random number.)
  • static void limitParams(dvector_t& params)
    Make sure that the parameters in the dvector_t aren't out of range. Adjust appropriately if any are.
    If the parameters in your function have no bounds, just define this with an empty body.

You'll notice that the FIT_FN class doesn't need to provide the function itself. This is by design, permitting FIT_FN to be compatible with the FORTRAN-based Levenberg-Marquardt minimizer, FitLM.

This class makes use of the drand48() pRNG. Call jpw_math::statistics::init_rand() before using this class.

const double WIDE_XOVER_WEIGHT
static

Widened crossover weight.

After about 10 generations, we start to get convergence <==> inbreeeding. To prevent this, we use a wider random weight in crossover, a weight which should be in the interval [1.0,1.5].
