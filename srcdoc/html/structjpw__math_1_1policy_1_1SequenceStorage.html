Detailed Description

template<class S>
struct jpw_math::policy::SequenceStorage< S >

Policy Class for how the native sequence is stored.

The SequenceStorage policy class templates provide a wrapper around the quirks of different sequential storage types. It takes a single template argument, S, which should be an STL sequence (like std::vector<...>), the boost::array<...> sequence or a POD-pointer array type.

All specializations of the SequenceStorage policy class template must define the following static member functions:

  • c create(size_t) provides an Adapter to different forms of S constructors. Specializations may have different return types.
  • destroy(ref_vec_t) will be a no-op for class types. It's only needed by POD-pointer array types to manage memory.
  • assign() will usually just call S::operator=(). For classes that do not perform a deep copy, specialize this function. Specializations for POD-pointer array types must manage memory as needed.
  • new_copy_of(constref_vec_t, size_t) should perform a deep-copy of the vector passed to it. The caller is expected to manage any memory returned to it (if it returns a pointer type, that is). It will usually just return via the copy-c'tor.
  • template<typename ITER> new_copy_of(ITER begin, ITER end) is an overloaded template function that should take two iterators defining the sequence to copy.
  • swap(ref_vec_t, ref_vec_t) function should call the native swap() member. If there is no native swap() member function in ref_vec_t, this function should be specialized to manually swap the two arguments.
