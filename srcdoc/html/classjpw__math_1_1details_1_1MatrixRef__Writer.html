Detailed Description

template<class C>
class jpw_math::details::MatrixRef_Writer< C >

Provides a means for optimized write-access to the underlying block of memory containing the elements of a Matrix<C>.

Use it for cross-language or C-API support.

An illustrative example will say it all:

fortran_matrix_invert_(double* m, int rr, int cc);
extern void
legacy_C_reader(someHandle_t h, double* m, size_t nn);

As this example shows, MatrixRef_Writer<C> takes the place of a non-const Matrix<C>& arg. The function MatrixRef_Writer::c_data(), returns a pointer to the block of memory where the Matrix<C> data starts. Notice how MatrixRef_Writer::c_data() forces you to specify the underlying Matrix object's dimensions. This is by design, since your legacy API will almost certainly require you to pass the dimensions to it, too.

You can also use this class with legacy APIs that read in matrix data as a "flattened" 1-D sequence. Just be sure to resize the underlying Matrix object so that its number of rows matches the size of the slowly-varying part of the "flattened" data.

Member Function Documentation

value_type* c_data ( size_t  row_size,
size_t  column_size 
)
inline

Returns a pointer to the underlying data stored in the Matrix.

The parameters row_size and column_size specify the target layout of the data you are storing in the memory address returned.

row_size must be the "slower moving" index, i.e. c_data(rsz,csz)+i*rsz+j is the address of m[i][j] . You cannot guarantee the integrity of the underlying Matrix object otherwise.

The underlying Matrix object will be resized to fit the specified dimensions, if necessary. Consider any data in the underlying Matrix object as lost after calling this function.

Use this function only with legacy C-APIs, as per Meyers' "Effective STL", and even then, only in concert with Matrix::size() on the underlying Matrix object.
